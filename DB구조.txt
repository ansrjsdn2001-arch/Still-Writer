Still Writer 데이터베이스 구조
==============================

작성 기준
---------
- 기준 DBMS: PostgreSQL
- 기본 키(PK): BIGINT GENERATED BY DEFAULT AS IDENTITY
- 날짜/시간: TIMESTAMP WITH TIME ZONE 권장
- 비밀번호와 인증 토큰은 원문이 아닌 해시값만 저장
- 글, 폴더, 소재는 deleted_at을 이용한 소프트 삭제 적용


1. 회원 테이블 (members)
-------------------------
회원의 기본 정보와 계정 상태를 관리합니다.

| 컬럼명            | 데이터 타입           | 필수 | 제약조건                    | 설명                         |
|-------------------|-----------------------|------|-----------------------------|------------------------------|
| member_id         | BIGINT                | O    | PK, 시퀀스                  | 회원 고유번호                |
| email             | VARCHAR(255)          | O    | UNIQUE                      | 로그인 이메일                |
| password_hash     | VARCHAR(255)          | X    |                             | 암호화된 비밀번호            |
| phone_number      | VARCHAR(20)           | X    | UNIQUE                      | 전화번호                     |
| nickname          | VARCHAR(50)           | O    | UNIQUE                      | 사용자 닉네임                |
| member_status     | VARCHAR(20)           | O    | DEFAULT 'ACTIVE'            | ACTIVE, SUSPENDED, WITHDRAWN |
| email_verified    | BOOLEAN               | O    | DEFAULT FALSE               | 이메일 인증 여부             |
| last_login_at     | TIMESTAMP WITH TIME ZONE | X |                             | 마지막 로그인 일시           |
| created_at        | TIMESTAMP WITH TIME ZONE | O | DEFAULT CURRENT_TIMESTAMP   | 가입일                       |
| updated_at        | TIMESTAMP WITH TIME ZONE | O | DEFAULT CURRENT_TIMESTAMP   | 회원정보 수정일              |
| withdrawn_at      | TIMESTAMP WITH TIME ZONE | X |                             | 회원 탈퇴일                  |

참고:
- password_hash에는 원문 비밀번호를 저장하지 않습니다.
- 소셜 로그인 전용 회원은 password_hash가 NULL일 수 있습니다.
- 이메일과 닉네임은 대소문자 정책을 정한 뒤 중복을 검사해야 합니다.


2. 소셜 로그인 계정 테이블 (social_accounts)
------------------------------------------------
카카오, 구글 등의 소셜 계정과 회원을 연결합니다.

| 컬럼명             | 데이터 타입              | 필수 | 제약조건                  | 설명                         |
|--------------------|--------------------------|------|---------------------------|------------------------------|
| social_account_id  | BIGINT                   | O    | PK, 시퀀스                | 소셜 계정 고유번호           |
| member_id          | BIGINT                   | O    | FK -> members.member_id   | 연결된 회원                  |
| provider           | VARCHAR(20)              | O    |                           | KAKAO, GOOGLE 등             |
| provider_user_id   | VARCHAR(255)             | O    |                           | 소셜 서비스 사용자 식별자    |
| provider_email     | VARCHAR(255)             | X    |                           | 소셜 서비스에서 받은 이메일  |
| created_at         | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 연동일                       |
| updated_at         | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 연동정보 수정일              |

고유 제약조건:
- UNIQUE(provider, provider_user_id)


3. 로그인 세션 테이블 (refresh_tokens)
-----------------------------------------
로그인 유지와 안전한 토큰 폐기를 위해 Refresh Token을 관리합니다.

| 컬럼명           | 데이터 타입              | 필수 | 제약조건                  | 설명                     |
|------------------|--------------------------|------|---------------------------|--------------------------|
| refresh_token_id | BIGINT                   | O    | PK, 시퀀스                | 토큰 고유번호            |
| member_id        | BIGINT                   | O    | FK -> members.member_id   | 토큰 소유 회원           |
| token_hash       | VARCHAR(255)             | O    | UNIQUE                    | Refresh Token 해시값     |
| device_info      | VARCHAR(500)             | X    |                           | 브라우저 및 기기 정보    |
| ip_address       | VARCHAR(45)              | X    |                           | IPv4 또는 IPv6 주소      |
| expires_at       | TIMESTAMP WITH TIME ZONE | O    |                           | 토큰 만료일              |
| revoked_at       | TIMESTAMP WITH TIME ZONE | X    |                           | 토큰 폐기일              |
| created_at       | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 토큰 발급일              |


4. 비밀번호 재설정 토큰 테이블 (password_reset_tokens)
----------------------------------------------------------
일회용 비밀번호 재설정 요청을 관리합니다.

| 컬럼명         | 데이터 타입              | 필수 | 제약조건                  | 설명                  |
|----------------|--------------------------|------|---------------------------|-----------------------|
| reset_token_id | BIGINT                   | O    | PK, 시퀀스                | 요청 고유번호         |
| member_id      | BIGINT                   | O    | FK -> members.member_id   | 요청 회원             |
| token_hash     | VARCHAR(255)             | O    | UNIQUE                    | 재설정 토큰 해시값    |
| expires_at     | TIMESTAMP WITH TIME ZONE | O    |                           | 토큰 만료일           |
| used_at        | TIMESTAMP WITH TIME ZONE | X    |                           | 토큰 사용일           |
| created_at     | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 요청일                |


5. 사용자 설정 테이블 (member_settings)
-------------------------------------------
회원별 편집기 및 화면 설정을 관리합니다.

| 컬럼명             | 데이터 타입              | 필수 | 제약조건                  | 설명                         |
|--------------------|--------------------------|------|---------------------------|------------------------------|
| member_id          | BIGINT                   | O    | PK, FK -> members.member_id | 설정 소유 회원             |
| theme              | VARCHAR(20)              | O    | DEFAULT 'SYSTEM'          | LIGHT, DARK, SYSTEM          |
| font_family        | VARCHAR(100)             | O    |                           | 편집기 글꼴                  |
| font_size          | SMALLINT                 | O    | DEFAULT 16                | 편집기 글자 크기             |
| line_height        | DECIMAL(3,1)             | O    | DEFAULT 1.6               | 편집기 줄 간격               |
| editor_width       | VARCHAR(20)              | O    | DEFAULT 'NORMAL'          | 편집 영역 너비               |
| auto_save_enabled  | BOOLEAN                  | O    | DEFAULT TRUE              | 자동 저장 여부               |
| auto_save_interval | INTEGER                  | O    | DEFAULT 30                | 자동 저장 간격(초)           |
| created_at         | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 설정 생성일                  |
| updated_at         | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 설정 수정일                  |


6. 글 폴더 테이블 (folders)
-----------------------------
소설, 일기, 메모 등의 글 폴더와 하위 폴더를 관리합니다.

| 컬럼명               | 데이터 타입              | 필수 | 제약조건                  | 설명                        |
|----------------------|--------------------------|------|---------------------------|-----------------------------|
| folder_id            | BIGINT                   | O    | PK, 시퀀스                | 폴더 고유번호               |
| member_id            | BIGINT                   | O    | FK -> members.member_id   | 폴더 소유 회원              |
| parent_folder_id     | BIGINT                   | X    | FK -> folders.folder_id   | 상위 폴더                   |
| folder_type          | VARCHAR(20)              | O    |                           | NOVEL, DIARY, MEMO          |
| folder_name          | VARCHAR(100)             | O    |                           | 폴더 이름                   |
| description          | VARCHAR(500)             | X    |                           | 폴더 설명                   |
| target_character_count | INTEGER                | X    |                           | 소설 등의 목표 글자 수      |
| sort_order           | INTEGER                  | O    | DEFAULT 0                 | 사용자 지정 정렬 순서       |
| created_at           | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 생성일                      |
| updated_at           | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 수정일                      |
| deleted_at           | TIMESTAMP WITH TIME ZONE | X    |                           | 휴지통 이동일               |

참고:
- parent_folder_id가 NULL이면 최상위 폴더입니다.
- deleted_at이 NULL이면 정상 상태이고, 값이 있으면 휴지통 상태입니다.


7. 작성 글 테이블 (documents)
-------------------------------
사용자가 작성한 글의 본문, 상태 및 통계 정보를 관리합니다.

| 컬럼명                        | 데이터 타입              | 필수 | 제약조건                  | 설명                          |
|-------------------------------|--------------------------|------|---------------------------|-------------------------------|
| document_id                   | BIGINT                   | O    | PK, 시퀀스                | 글 고유번호                   |
| member_id                     | BIGINT                   | O    | FK -> members.member_id   | 글 작성자                     |
| folder_id                     | BIGINT                   | X    | FK -> folders.folder_id   | 글이 속한 폴더                |
| title                         | VARCHAR(200)             | O    |                           | 글 제목                       |
| content                       | TEXT                     | O    | DEFAULT ''                | HTML 또는 에디터 JSON 본문    |
| plain_text                    | TEXT                     | O    | DEFAULT ''                | 서식이 제거된 검색용 본문     |
| character_count               | INTEGER                  | O    | DEFAULT 0                 | 공백 포함 글자 수             |
| character_count_without_spaces| INTEGER                  | O    | DEFAULT 0                 | 공백 제외 글자 수             |
| word_count                    | INTEGER                  | O    | DEFAULT 0                 | 단어 수                       |
| document_status               | VARCHAR(20)              | O    | DEFAULT 'DRAFT'           | DRAFT, COMPLETED              |
| last_opened_at                | TIMESTAMP WITH TIME ZONE | X    |                           | 마지막으로 연 시간            |
| created_at                    | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 최초 작성일                   |
| updated_at                    | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 마지막 수정일                 |
| deleted_at                    | TIMESTAMP WITH TIME ZONE | X    |                           | 휴지통 이동일                 |

plain_text 사용 목적:
- content가 HTML 또는 JSON일 때 태그와 서식을 제거한 순수 텍스트를 저장합니다.
- 본문 검색, 검색 결과 미리보기, 글자 수 계산에 사용합니다.
- 클라이언트가 전달한 값을 그대로 저장하지 않고 백엔드에서 content를 기준으로 생성합니다.
- content를 일반 텍스트로만 저장한다면 plain_text 컬럼은 제거할 수 있습니다.


8. 즐겨찾기 테이블 (document_favorites)
------------------------------------------
회원이 즐겨찾기로 등록한 글을 관리합니다.

| 컬럼명      | 데이터 타입              | 필수 | 제약조건                    | 설명                |
|-------------|--------------------------|------|-----------------------------|---------------------|
| favorite_id | BIGINT                   | O    | PK, 시퀀스                  | 즐겨찾기 고유번호   |
| member_id   | BIGINT                   | O    | FK -> members.member_id     | 즐겨찾기 설정 회원  |
| document_id | BIGINT                   | O    | FK -> documents.document_id | 즐겨찾기한 글       |
| created_at  | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP   | 등록일              |

고유 제약조건:
- UNIQUE(member_id, document_id)


9. 소재 보관함 테이블 (materials)
------------------------------------
소재 폴더, 업로드 파일, 외부 링크 및 텍스트 소재를 계층형으로 관리합니다.

| 컬럼명             | 데이터 타입              | 필수 | 제약조건                    | 설명                         |
|--------------------|--------------------------|------|-----------------------------|------------------------------|
| material_id        | BIGINT                   | O    | PK, 시퀀스                  | 소재 고유번호                |
| member_id          | BIGINT                   | O    | FK -> members.member_id     | 소재 소유 회원               |
| parent_material_id | BIGINT                   | X    | FK -> materials.material_id | 상위 소재 폴더               |
| material_type      | VARCHAR(20)              | O    |                             | FOLDER, FILE, LINK, TEXT     |
| material_name      | VARCHAR(255)             | O    |                             | 소재 또는 폴더 이름          |
| text_content       | TEXT                     | X    |                             | 텍스트 소재 내용             |
| original_file_name | VARCHAR(255)             | X    |                             | 업로드 당시 파일명           |
| storage_key        | VARCHAR(500)             | X    |                             | 서버 또는 스토리지 저장 키   |
| external_url       | VARCHAR(2048)            | X    |                             | 외부 링크 주소               |
| mime_type          | VARCHAR(100)             | X    |                             | 파일 MIME 타입               |
| file_size          | BIGINT                   | X    |                             | 파일 크기(바이트)            |
| sort_order         | INTEGER                  | O    | DEFAULT 0                   | 사용자 지정 정렬 순서        |
| created_at         | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP   | 생성일                       |
| updated_at         | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP   | 수정일                       |
| deleted_at         | TIMESTAMP WITH TIME ZONE | X    |                             | 휴지통 이동일                |

참고:
- parent_material_id가 NULL이면 소재 보관함 최상위 항목입니다.
- 사용자 PC의 절대 경로 대신 서버나 객체 스토리지의 storage_key를 저장합니다.


10. 글-소재 연결 테이블 (document_materials)
------------------------------------------------
글과 소재의 다대다(N:M) 관계를 관리합니다.

| 컬럼명              | 데이터 타입              | 필수 | 제약조건                    | 설명             |
|---------------------|--------------------------|------|-----------------------------|------------------|
| document_material_id| BIGINT                   | O    | PK, 시퀀스                  | 연결 고유번호    |
| document_id         | BIGINT                   | O    | FK -> documents.document_id | 연결된 글        |
| material_id         | BIGINT                   | O    | FK -> materials.material_id | 연결된 소재      |
| created_at          | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP   | 연결일           |

고유 제약조건:
- UNIQUE(document_id, material_id)


11. 일별 집필 기록 테이블 (daily_writing_stats)
-------------------------------------------------
연속 작성일, 일별 작성량 및 집필 통계를 관리합니다.

| 컬럼명                  | 데이터 타입              | 필수 | 제약조건                  | 설명                      |
|-------------------------|--------------------------|------|---------------------------|---------------------------|
| writing_stat_id         | BIGINT                   | O    | PK, 시퀀스                | 집필 기록 고유번호        |
| member_id               | BIGINT                   | O    | FK -> members.member_id   | 작성 회원                 |
| writing_date            | DATE                     | O    |                           | 집필 날짜                 |
| written_character_count | INTEGER                  | O    | DEFAULT 0                 | 해당 날짜에 작성한 글자 수|
| writing_seconds         | INTEGER                  | O    | DEFAULT 0                 | 해당 날짜의 집필 시간(초) |
| created_at              | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 기록 생성일               |
| updated_at              | TIMESTAMP WITH TIME ZONE | O    | DEFAULT CURRENT_TIMESTAMP | 기록 수정일               |

고유 제약조건:
- UNIQUE(member_id, writing_date)


테이블 관계 요약
----------------
| 부모 테이블 | 관계 | 자식 테이블                    |
|-------------|------|--------------------------------|
| members     | 1:N  | social_accounts                |
| members     | 1:N  | refresh_tokens                 |
| members     | 1:N  | password_reset_tokens          |
| members     | 1:1  | member_settings                |
| members     | 1:N  | folders                        |
| members     | 1:N  | documents                      |
| members     | 1:N  | materials                      |
| members     | 1:N  | daily_writing_stats            |
| folders     | 1:N  | documents                      |
| folders     | 1:N  | 하위 folders                   |
| documents   | N:M  | materials(document_materials)  |
| documents   | 1:N  | document_favorites             |
| materials   | 1:N  | 하위 materials                 |


삭제 정책
---------
| 대상      | 삭제 방식   | 처리 방법                                      |
|-----------|-------------|------------------------------------------------|
| 글        | 소프트 삭제 | documents.deleted_at에 삭제 시각 기록          |
| 글 폴더   | 소프트 삭제 | folders.deleted_at에 삭제 시각 기록            |
| 소재      | 소프트 삭제 | materials.deleted_at에 삭제 시각 기록          |
| 휴지통 복원 | 삭제 취소 | deleted_at을 NULL로 변경                       |
| 영구 삭제 | 물리 삭제   | 보존 기간이 지난 후 실제 DELETE 수행           |
| 회원 탈퇴 | 소프트 삭제 | member_status를 WITHDRAWN으로 변경하고 탈퇴일 기록 |


권장 인덱스
-----------
- members(email)
- members(nickname)
- folders(member_id, deleted_at)
- folders(parent_folder_id)
- documents(member_id, deleted_at, updated_at DESC)
- documents(folder_id, deleted_at)
- documents(title)
- document_favorites(member_id, created_at DESC)
- materials(member_id, parent_material_id, deleted_at)
- daily_writing_stats(member_id, writing_date DESC)

검색 기능을 본격적으로 구현할 때는 documents.plain_text에 PostgreSQL 전문 검색
(GIN 인덱스 또는 별도 tsvector 컬럼)을 적용하는 것을 권장합니다.
