Still Writer 데이터베이스 구조
==============================

작성 기준
---------
- 기준 DBMS: MySQL 8.0 이상
- 문자 집합: utf8mb4
- 기본 키(PK): BIGINT UNSIGNED AUTO_INCREMENT
- 날짜/시간: DATETIME(6), 애플리케이션과 DB는 UTC 저장 권장
- 비밀번호와 인증 토큰은 원문이 아닌 해시값만 저장
- Google Drive OAuth 토큰은 애플리케이션 암호화 후 VARBINARY로 저장
- 글, 폴더, 소재는 deleted_at과 purge_after를 이용한 소프트 삭제 적용
- 수동 저장 버튼은 사용하지 않으며 글 작성 내용은 자동 저장


1. 회원 테이블 (users)
----------------------
회원의 기본 정보와 계정 상태를 관리합니다.

| 컬럼명            | 데이터 타입                                  | 필수 | 제약조건                                      | 설명                         |
|-------------------|----------------------------------------------|------|-----------------------------------------------|------------------------------|
| id                | BIGINT UNSIGNED                              | O    | PK, AUTO_INCREMENT                            | 회원 고유번호                |
| email             | VARCHAR(191)                                 | O    | UNIQUE                                        | 로그인 이메일                |
| password_hash     | VARCHAR(255)                                 | X    |                                               | 해시 처리된 비밀번호         |
| nickname          | VARCHAR(50)                                  | O    | UNIQUE                                        | 사용자 닉네임                |
| profile_image_url | VARCHAR(2048)                                | X    |                                               | 프로필 이미지 주소           |
| email_verified_at | DATETIME(6)                                  | X    |                                               | 이메일 인증 완료 일시        |
| status            | ENUM('ACTIVE','SUSPENDED','WITHDRAWN')       | O    | DEFAULT 'ACTIVE'                              | 계정 상태                    |
| last_login_at     | DATETIME(6)                                  | X    |                                               | 마지막 로그인 일시           |
| created_at        | DATETIME(6)                                  | O    | DEFAULT CURRENT_TIMESTAMP(6)                  | 가입일                       |
| updated_at        | DATETIME(6)                                  | O    | DEFAULT CURRENT_TIMESTAMP(6), 갱신 시 업데이트 | 회원정보 수정일              |
| withdrawn_at      | DATETIME(6)                                  | X    |                                               | 회원 탈퇴일                  |

참고:
- 소셜 로그인 전용 회원은 password_hash가 NULL일 수 있습니다.
- 이메일은 소문자로 정규화한 뒤 중복을 검사합니다.
- 회원 탈퇴 시 status를 WITHDRAWN으로 변경하고 withdrawn_at을 기록합니다.


2. 인증 계정 테이블 (auth_identities)
--------------------------------------
일반, Google, Kakao 로그인 계정을 회원과 연결합니다.

| 컬럼명            | 데이터 타입                       | 필수 | 제약조건                         | 설명                      |
|-------------------|-----------------------------------|------|----------------------------------|---------------------------|
| id                | BIGINT UNSIGNED                   | O    | PK, AUTO_INCREMENT               | 인증 계정 고유번호        |
| user_id           | BIGINT UNSIGNED                   | O    | FK -> users.id                   | 연결된 회원               |
| provider          | ENUM('LOCAL','GOOGLE','KAKAO')    | O    |                                  | 로그인 제공자             |
| provider_user_id  | VARCHAR(191)                      | O    |                                  | 제공자의 사용자 식별자    |
| provider_email    | VARCHAR(191)                      | X    |                                  | 제공자에서 받은 이메일    |
| created_at        | DATETIME(6)                       | O    | DEFAULT CURRENT_TIMESTAMP(6)     | 연동일                    |
| updated_at        | DATETIME(6)                       | O    | DEFAULT CURRENT_TIMESTAMP(6)     | 연동정보 수정일           |

고유 제약조건:
- UNIQUE(provider, provider_user_id)
- UNIQUE(user_id, provider)


3. 로그인 세션 테이블 (user_sessions)
---------------------------------------
로그인 유지와 안전한 토큰 폐기를 위해 Refresh Token을 관리합니다.

| 컬럼명            | 데이터 타입       | 필수 | 제약조건                     | 설명                    |
|-------------------|-------------------|------|------------------------------|-------------------------|
| id                | BIGINT UNSIGNED   | O    | PK, AUTO_INCREMENT           | 세션 고유번호           |
| user_id           | BIGINT UNSIGNED   | O    | FK -> users.id               | 세션 소유 회원          |
| refresh_token_hash| CHAR(64)          | O    | UNIQUE                       | Refresh Token 해시값    |
| device_name       | VARCHAR(100)      | X    |                              | 기기 이름               |
| ip_address        | VARBINARY(16)     | X    |                              | IPv4 또는 IPv6 주소     |
| user_agent        | VARCHAR(500)      | X    |                              | 브라우저 및 기기 정보   |
| expires_at        | DATETIME(6)       | O    |                              | 토큰 만료일             |
| revoked_at        | DATETIME(6)       | X    |                              | 토큰 폐기일             |
| created_at        | DATETIME(6)       | O    | DEFAULT CURRENT_TIMESTAMP(6) | 토큰 발급일             |


4. 비밀번호 재설정 토큰 테이블 (password_reset_tokens)
-------------------------------------------------------
일회용 비밀번호 재설정 요청을 관리합니다.

| 컬럼명     | 데이터 타입       | 필수 | 제약조건                     | 설명              |
|------------|-------------------|------|------------------------------|-------------------|
| id         | BIGINT UNSIGNED   | O    | PK, AUTO_INCREMENT           | 요청 고유번호     |
| user_id    | BIGINT UNSIGNED   | O    | FK -> users.id               | 요청 회원         |
| token_hash | CHAR(64)          | O    | UNIQUE                       | 재설정 토큰 해시값|
| expires_at | DATETIME(6)       | O    |                              | 토큰 만료일       |
| used_at    | DATETIME(6)       | X    |                              | 토큰 사용일       |
| created_at | DATETIME(6)       | O    | DEFAULT CURRENT_TIMESTAMP(6) | 요청일            |


5. 사용자 설정 테이블 (user_settings)
---------------------------------------
회원별 편집기 및 화면 설정을 관리합니다.

| 컬럼명          | 데이터 타입                                | 필수 | 제약조건                     | 설명                         |
|-----------------|--------------------------------------------|------|------------------------------|------------------------------|
| user_id         | BIGINT UNSIGNED                            | O    | PK, FK -> users.id           | 설정 소유 회원               |
| theme           | ENUM('SYSTEM','LIGHT','DARK')              | O    | DEFAULT 'SYSTEM'             | 화면 테마                    |
| font_family     | VARCHAR(100)                               | O    | DEFAULT 'SYSTEM'             | 편집기 글꼴                  |
| font_size       | TINYINT UNSIGNED                           | O    | DEFAULT 16                   | 편집기 글자 크기             |
| line_height     | DECIMAL(3,1)                               | O    | DEFAULT 1.6                  | 편집기 줄 간격               |
| paragraph_gap   | ENUM('NARROW','NORMAL','WIDE')             | O    | DEFAULT 'NORMAL'             | 문단 간격                    |
| auto_indent     | BOOLEAN                                    | O    | DEFAULT TRUE                 | 자동 들여쓰기 사용 여부      |
| char_count_mode | ENUM('INCLUDE_SPACES','EXCLUDE_SPACES')    | O    | DEFAULT 'INCLUDE_SPACES'     | 글자 수 표시 기준            |
| export_format   | ENUM('TXT','DOCX','PDF','MARKDOWN')        | O    | DEFAULT 'TXT'                | 기본 내보내기 형식           |
| locale          | VARCHAR(10)                                | O    | DEFAULT 'ko-KR'              | 언어 설정                    |
| timezone        | VARCHAR(50)                                | O    | DEFAULT 'Asia/Seoul'         | 사용자 시간대                |
| created_at      | DATETIME(6)                                | O    | DEFAULT CURRENT_TIMESTAMP(6) | 설정 생성일                  |
| updated_at      | DATETIME(6)                                | O    | DEFAULT CURRENT_TIMESTAMP(6) | 설정 수정일                  |

참고:
- 자동 저장은 항상 활성화하므로 사용자가 끄는 설정은 저장하지 않습니다.
- 자동 저장 주기는 서버 설정으로 관리합니다.


6. 외부 저장소 계정 테이블 (external_storage_accounts)
---------------------------------------------------------
Google Drive 계정과 OAuth 토큰 정보를 관리합니다.

| 컬럼명                | 데이터 타입                 | 필수 | 제약조건                     | 설명                         |
|-----------------------|-----------------------------|------|------------------------------|------------------------------|
| id                    | BIGINT UNSIGNED             | O    | PK, AUTO_INCREMENT           | 외부 저장소 계정 고유번호    |
| user_id               | BIGINT UNSIGNED             | O    | FK -> users.id               | 연동 회원                    |
| provider              | ENUM('GOOGLE_DRIVE')        | O    |                              | 저장소 제공자                |
| provider_account_id   | VARCHAR(191)                | O    |                              | 제공자의 계정 식별자         |
| account_email         | VARCHAR(191)                | X    |                              | 연동된 계정 이메일           |
| access_token_encrypted| VARBINARY(4096)             | X    |                              | 암호화된 Access Token        |
| refresh_token_encrypted| VARBINARY(4096)            | X    |                              | 암호화된 Refresh Token       |
| token_expires_at      | DATETIME(6)                 | X    |                              | Access Token 만료일          |
| created_at            | DATETIME(6)                 | O    | DEFAULT CURRENT_TIMESTAMP(6) | 연동일                       |
| updated_at            | DATETIME(6)                 | O    | DEFAULT CURRENT_TIMESTAMP(6) | 연동정보 수정일              |
| disconnected_at       | DATETIME(6)                 | X    |                              | 연동 해제일                  |

고유 제약조건:
- UNIQUE(user_id, provider, provider_account_id)

보안:
- OAuth 토큰 암호화 키는 DB가 아닌 AWS Secrets Manager 또는 환경 변수로 관리합니다.
- 토큰 값은 로그, 응답 데이터, 오류 메시지에 출력하지 않습니다.


7. 글 폴더 테이블 (folders)
----------------------------
소설, 일기, 메모 등의 글 폴더와 하위 폴더를 관리합니다.

| 컬럼명          | 데이터 타입                              | 필수 | 제약조건                     | 설명                       |
|-----------------|------------------------------------------|------|------------------------------|----------------------------|
| id              | BIGINT UNSIGNED                          | O    | PK, AUTO_INCREMENT           | 폴더 고유번호              |
| user_id         | BIGINT UNSIGNED                          | O    | FK -> users.id               | 폴더 소유 회원             |
| parent_id       | BIGINT UNSIGNED                          | X    | FK -> folders.id             | 상위 폴더                  |
| name            | VARCHAR(100)                             | O    |                              | 폴더 이름                  |
| description     | VARCHAR(500)                             | X    |                              | 폴더 설명                  |
| category        | ENUM('NOVEL','DIARY','MEMO','GENERAL')   | O    | DEFAULT 'GENERAL'            | 폴더 카테고리              |
| color           | CHAR(7)                                  | X    |                              | 폴더 색상                  |
| target_char_count| INT UNSIGNED                            | X    |                              | 소설 목표 글자 수          |
| diary_auto_date | BOOLEAN                                  | O    | DEFAULT FALSE                | 일기 날짜 자동 입력 여부   |
| sort_order      | INT                                      | O    | DEFAULT 0                    | 사용자 지정 정렬 순서      |
| last_used_at    | DATETIME(6)                              | X    |                              | 마지막 사용 일시           |
| created_at      | DATETIME(6)                              | O    | DEFAULT CURRENT_TIMESTAMP(6) | 생성일                     |
| updated_at      | DATETIME(6)                              | O    | DEFAULT CURRENT_TIMESTAMP(6) | 수정일                     |
| deleted_at      | DATETIME(6)                              | X    |                              | 휴지통 이동일              |
| purge_after     | DATETIME(6)                              | X    |                              | 영구 삭제 예정일           |

고유 제약조건:
- UNIQUE(user_id, parent_id, name, deleted_at) 적용 여부는 중복 폴더명 정책 확정 후 결정합니다.


8. 작성 글 테이블 (documents)
------------------------------
사용자가 작성한 글의 본문, 상태 및 통계 정보를 관리합니다.

| 컬럼명                   | 데이터 타입                                  | 필수 | 제약조건                     | 설명                         |
|--------------------------|----------------------------------------------|------|------------------------------|------------------------------|
| id                       | BIGINT UNSIGNED                              | O    | PK, AUTO_INCREMENT           | 글 고유번호                  |
| user_id                  | BIGINT UNSIGNED                              | O    | FK -> users.id               | 글 작성자                    |
| folder_id                | BIGINT UNSIGNED                              | X    | FK -> folders.id             | 글이 속한 폴더               |
| type                     | ENUM('NOVEL','DIARY','MEMO','GENERAL')       | O    | DEFAULT 'GENERAL'            | 글 유형                      |
| title                    | VARCHAR(50)                                  | O    | DEFAULT ''                   | 글 제목                      |
| content_json             | JSON                                         | O    |                              | 에디터 본문 데이터           |
| plain_text               | LONGTEXT                                     | O    |                              | 검색 및 미리보기용 본문      |
| char_count               | INT UNSIGNED                                 | O    | DEFAULT 0                    | 공백 포함 글자 수            |
| char_count_without_spaces| INT UNSIGNED                                 | O    | DEFAULT 0                    | 공백 제외 글자 수            |
| word_count               | INT UNSIGNED                                 | O    | DEFAULT 0                    | 단어 수                      |
| sentence_count           | INT UNSIGNED                                 | O    | DEFAULT 0                    | 문장 수                      |
| target_char_count        | INT UNSIGNED                                 | X    |                              | 글 단위 목표 글자 수         |
| diary_date               | DATE                                         | X    |                              | 일기 날짜                    |
| status                   | ENUM('DRAFT','IN_PROGRESS','COMPLETED')      | O    | DEFAULT 'DRAFT'              | 작성 상태                    |
| completed_at             | DATETIME(6)                                  | X    |                              | 작성 완료 일시               |
| last_opened_at           | DATETIME(6)                                  | X    |                              | 마지막으로 연 일시           |
| created_at               | DATETIME(6)                                  | O    | DEFAULT CURRENT_TIMESTAMP(6) | 최초 작성일                  |
| updated_at               | DATETIME(6)                                  | O    | DEFAULT CURRENT_TIMESTAMP(6) | 마지막 수정일                |
| deleted_at               | DATETIME(6)                                  | X    |                              | 휴지통 이동일                |
| purge_after              | DATETIME(6)                                  | X    |                              | 영구 삭제 예정일             |

참고:
- content_json, plain_text, 각종 글자 수는 백엔드가 동일한 본문을 기준으로 생성합니다.
- title은 화면 요구사항에 맞춰 최대 50자로 제한합니다.
- 제목을 입력하지 않은 자동 저장 글은 빈 문자열로 저장할 수 있습니다.


9. 글 자동 저장 버전 테이블 (document_versions)
--------------------------------------------------
자동 저장 시점의 글 버전을 보관하여 복구 기능을 지원합니다.

| 컬럼명       | 데이터 타입                 | 필수 | 제약조건                         | 설명                   |
|--------------|-----------------------------|------|----------------------------------|------------------------|
| id           | BIGINT UNSIGNED             | O    | PK, AUTO_INCREMENT               | 버전 고유번호          |
| document_id  | BIGINT UNSIGNED             | O    | FK -> documents.id               | 대상 글                |
| version_no   | INT UNSIGNED                | O    |                                  | 글별 버전 번호         |
| title        | VARCHAR(50)                 | O    | DEFAULT ''                       | 버전 제목              |
| content_json | JSON                        | O    |                                  | 버전 본문 데이터       |
| plain_text   | LONGTEXT                    | O    |                                  | 버전 일반 텍스트       |
| char_count   | INT UNSIGNED                | O    | DEFAULT 0                        | 공백 포함 글자 수      |
| version_type | ENUM('AUTO','RESTORE')      | O    | DEFAULT 'AUTO'                   | 자동 저장 또는 복구본  |
| created_at   | DATETIME(6)                 | O    | DEFAULT CURRENT_TIMESTAMP(6)     | 버전 생성일            |

고유 제약조건:
- UNIQUE(document_id, version_no)

참고:
- 수동 저장 버튼을 사용하지 않으므로 MANUAL 유형은 정의하지 않습니다.
- 버전 보관 개수 또는 보관 기간을 정해 오래된 버전을 정리합니다.

13. 소재 보관함 테이블 (materials)
------------------------------------
기존 글, 업로드 파일, 이미지를 글쓰기 소재로 관리합니다.

| 컬럼명            | 데이터 타입                                  | 필수 | 제약조건                     | 설명                         |
|-------------------|----------------------------------------------|------|------------------------------|------------------------------|
| id                | BIGINT UNSIGNED                              | O    | PK, AUTO_INCREMENT           | 소재 고유번호                |
| user_id           | BIGINT UNSIGNED                              | O    | FK -> users.id               | 소재 소유 회원               |
| source_document_id| BIGINT UNSIGNED                              | X    | FK -> documents.id           | 소재의 원본 글               |
| type              | ENUM('DOCUMENT','FILE','IMAGE')              | O    |                              | 소재 유형                    |
| title             | VARCHAR(255)                                 | O    |                              | 소재 제목                    |
| description       | VARCHAR(1000)                                | X    |                              | 소재 설명                    |
| content_snapshot  | JSON                                         | X    |                              | 등록 시점의 글 내용          |
| original_file_name| VARCHAR(255)                                 | X    |                              | 업로드 당시 파일명           |
| storage_provider  | ENUM('APP_STORAGE','GOOGLE_DRIVE','S3')      | X    |                              | 파일 저장소                  |
| storage_key       | VARCHAR(1024)                                | X    |                              | 서버 또는 객체 저장소 키     |
| mime_type         | VARCHAR(191)                                 | X    |                              | 파일 MIME 타입               |

검색 정책
---------
- 기본 검색 대상은 documents.title, documents.plain_text, folders.name입니다.
- MySQL FULLTEXT 인덱스는 한글 검색 품질과 최소 토큰 길이를 검증한 뒤 적용합니다.
- 초기 구현은 사용자 범위를 먼저 제한한 뒤 제목과 본문을 검색합니다.
- 데이터 증가 시 OpenSearch 같은 별도 검색 엔진 도입을 검토합니다.


검증 규칙
---------
- title은 50자 이하인지 백엔드에서 검사합니다.
- type과 folder.category 조합이 유효한지 백엔드에서 검사합니다.
- target_char_count는 NOVEL 유형에서만 사용합니다.
- diary_date는 DIARY 유형에서만 사용합니다.
- 소재 파일의 MIME 타입, 확장자, 크기를 서버에서 검사합니다.
- 모든 조회와 변경 요청에서 user_id 소유권을 검사합니다.
- content_json에서 plain_text와 글자 수 통계를 백엔드가 다시 계산합니다.
