# Still Writer 트러블슈팅

## 로그인 페이지의 테마별 히어로 이미지 분리

### 문제 상황

초기에는 로그인 페이지에서 하나의 히어로 이미지만 사용했습니다. 그러나 해당 이미지가 밝은 색상 위주로 구성되어 있어, 다크 모드를 사용하는 사용자가 히어로 이미지가 있는 페이지에 진입할 때 화면이 지나치게 밝게 느껴지고 눈의 피로를 유발할 수 있다고 판단했습니다.

### 해결 방법

라이트 모드와 다크 모드에 각각 적합한 전용 히어로 이미지를 별도로 제작하고, 현재 앱 테마에 따라 자동으로 전환되도록 적용했습니다.

- 라이트 모드: `frontend/public/images/heroimage(light).png`
- 다크 모드: `frontend/public/images/heroimage(dark).png`
- 적용 화면: 로그인 페이지
- 테마 판별 기준: `.app` 요소의 `data-theme` 속성

### 적용 결과

다크 모드 사용자가 로그인 페이지에 진입했을 때 밝은 이미지로 인한 시각적 부담이 줄었으며, 각 테마의 전체적인 색감과 로그인 화면의 배경 이미지가 자연스럽게 어울리도록 개선했습니다.
